# 510K 固定战术测试矩阵

本矩阵用于约束规则 AI、模型训练、正式评测、机器人、托管和提示的一致行为。用例只使用玩家可见信息；“应优先”用于 Top-1 门禁，“必须存在”用于合法候选门禁，“不得”用于反例门禁。

## 一、规则与合法动作

| 编号 | 局面 | 预期 |
|---|---|---|
| R01-R13 | 单张、对子、三带一、三带一对、顺子、连对、飞机、四带二单、四带二对、假510K、真510K、普通炸弹、王炸 | 每种完整牌型都必须生成且识别正确 |
| R14 | 经典模式出现 5/10/K | 不得识别为 510K |
| R15 | 跟牌阶段无法压制 | 必须包含“不出” |
| R16 | 新牌墩首出 | 不得包含“不出” |
| R17 | 相同牌型、相同长度、点数更大 | 允许压制 |
| R18 | 顺子或连对长度不同 | 不允许压制 |
| R19 | 同级真510K或同级假510K | 互相不能压制 |
| R20 | 普通牌、假510K、普通炸弹、真510K、王炸 | 严格遵守既定五层压制顺序 |

## 二、首出与减手数

| 编号 | 局面 | 预期 |
|---|---|---|
| L01 | 仅有 3、10、2 三张散牌 | 优先出 3，保留控制牌和 10 的后压价值 |
| L02 | 小对子加两张散牌 | 优先完整小对子，不拆成单张 |
| L03 | 完整顺子加一张大单 | 优先顺子 |
| L04 | 完整连对加一张大单 | 优先连对 |
| L05 | 三条、小单、大对子 | 优先三带小单，保留大对子 |
| L06 | 三条、两个单张 | 选择较小、较难独立走掉的翅膀 |
| L07 | 三条、小对子、大对子 | 若手数相同，优先保留大对子 |
| L08 | 飞机和散牌 | 优先完整飞机或合法带翅膀方案，不拆连续三条 |
| L09 | 真510K加小散牌 | 非直接跑完时优先处理小散牌 |
| L10 | 假510K加小散牌 | 非直接跑完时优先处理小散牌 |
| L11 | 普通炸弹加可正常走的小牌 | 不得无意义先炸 |
| L12 | 王炸加普通牌 | 不得无意义先王炸 |
| L13 | 任意完整合法牌型恰好是全部手牌 | 必须直接出完 |
| L14 | 对手只剩一张，自己有对子和大单 | 优先对手无法跟的对子夺取连续牌权 |
| L15 | 对手只剩一张，自己全为散牌 | 综合公开历史选择控制牌，不固定机械出最大牌 |

## 三、跟牌与拆牌成本

| 编号 | 局面 | 预期 |
|---|---|---|
| F01 | 多张单牌都能压住 | 使用最小的可压单牌 |
| F02 | 多个对子都能压住 | 使用最小的可压对子 |
| F03 | 多个三条都能压住 | 使用最小的可压三条 |
| F04 | 多个同长度顺子都能压住 | 使用最小的可压顺子 |
| F05 | 有独立单张和可拆对子都能压 | 优先独立单张 |
| F06 | 有完整对子和可拆三条都能压 | 优先完整对子 |
| F07 | 有普通响应和炸弹都能压 | 优先普通响应 |
| F08 | 有普通响应和真/假510K都能压 | 优先普通响应 |
| F09 | 只能拆顺子才能压且桌面无分、对手不听牌 | 允许不出，保留顺子 |
| F10 | 拆顺子能直接赢下整局 | 允许拆顺子 |
| F11 | 拆对子能阻止下一家直接出完 | 允许拆对子 |
| F12 | 跟牌后自己可在下一手直接跑完 | 提高该响应优先级 |

## 四、炸弹与 510K 使用

| 编号 | 局面 | 预期 |
|---|---|---|
| B01 | 只有炸弹可压、桌面无分、对手不听牌 | 优先不出，保留炸弹 |
| B02 | 炸弹是全部手牌 | 直接炸完 |
| B03 | 对手明确只剩一手且炸弹可阻止 | 提高炸弹优先级 |
| B04 | 桌面有高分但仍有普通响应 | 不因分数固定阈值强制炸弹 |
| B05 | 四张 K 加两个大对子和其他牌 | 不得首选四带二对拆掉 K 炸弹 |
| B06 | 四带二单可一次直接出完 | 允许并优先 |
| B07 | 四带二对可一次直接出完 | 允许并优先 |
| B08 | 假510K可压普通牌，但对手无威胁 | 不得机械消耗 |
| B09 | 真510K可压普通炸弹并阻止对手出完 | 应纳入高优先候选 |
| B10 | 王炸可赢当前墩但会破坏后续跑牌 | 由终局价值决定，不设固定“必炸”条件 |

## 五、分牌、罚分与牌权

| 编号 | 局面 | 预期 |
|---|---|---|
| S01 | 出单张 5、10、K | 公开分值分别按 5、10、20 编码 |
| S02 | 一副牌全部分牌 | 总分严格为 140 |
| S03 | 自己领先且手数少 | 首先保证跑完，不为小分破坏牌形 |
| S04 | 胜率相近且桌面高分 | 倾向争夺牌权和吃分 |
| S05 | 手牌剩余大量 5/10/K | 动作特征必须体现剩余罚分风险 |
| S06 | K 炸弹留在输家手中 | 终局按 80 分罚分进入训练回报 |
| S07 | 5/10/K 可作为低代价翅膀出掉 | 比保留到终局更有价值，但不能压过必胜动作 |
| S08 | 抢到牌墩后可连续出完整组合 | 牌权收益应高于孤立吃一次小分 |

## 六、对手推断、残局和隐藏信息

| 编号 | 局面 | 预期 |
|---|---|---|
| E01 | 对手剩 1 张 | 重点防单张放走 |
| E02 | 对手剩 2 张 | 优先考虑对子封堵，也要考虑两张散牌历史 |
| E03 | 对手剩 3 张 | 结合历史判断单张、对子加单张或 510K，不固定必炸 |
| E04 | 最近多次不出单张 | 提高其缺少该点数以上单牌的概率特征，不直接窥牌 |
| E05 | 最近多次不出对子 | 提高对子牌型夺权价值 |
| E06 | 三家累计得分差异明显 | 区分领先跑牌和落后抢分策略 |
| E07 | 同一公开状态、交换服务器内两家暗牌 | 模型输出必须不变 |
| E08 | 最近动作超过 15 次 | 只保留最后 15 次且座次、出牌/不出标记正确 |
| E09 | 同一副牌评测 | 模型依次轮换三个座位，消除座位和发牌偏差 |
| E10 | 候选很多 | 全部合法动作都必须进入模型，不得先由规则截断 |

## 七、可靠性与跨链路一致性

| 编号 | 局面 | 预期 |
|---|---|---|
| Q01 | 模型文件缺失或版本不兼容 | 降级规则 AI，仍返回合法动作 |
| Q02 | 模型推理超时、异常或非法输出 | 降级规则 AI，不得卡住回合 |
| Q03 | 机器人、托管、提示面对同一公开状态 | 使用同一候选排序来源 |
| Q04 | Windows 两个 Actor 训练 | 可序列化、无 `BrokenProcessPool` |
| Q05 | 训练轨迹终局 | 三座位每次实际选择都绑定本人的相对终局回报 |
| Q06 | 旧 `public_state_v2` 权重 | 明确拒载，不允许静默错维度加载 |
| Q07 | 新训练清单 | 必须包含训练局数、特征版本、规则版本和 SHA-256 |
| Q08 | 正式上线门禁 | 胜率不低于 40%、得分差不低于 0、非法率为 0，并通过固定战术集 |
