# 2026-07-10 协同文档解耦与说明书升级设计规格书

本规格书定义了项目协作文档 `AGENTS.md` 与主说明书 `README.md` 的去冗余与升级设计。

## 1. 设计意图
解决 `AGENTS.md` 和 `README.md` 存在大量项目背景与命令冗余的问题，落实“单一事实来源（SSOT）”原则。
- **AGENTS.md**：定位为纯粹的“Agent 操作与协作规则限制集”，不再包含任何重复的技术细节。
- **README.md**：定位为项目唯一的“详尽说明书大门”，汇总所有 REST API 地图、WebSocket 长连接协议、分布式灾备高可用架构与 Conda 专用环境命令，且不缺失任何历史 HMP 能力介绍。

## 2. 文档结构定义

### 2.1 AGENTS.md 结构
- **保留章节**：`## 强制约束`、`## 智能体协作与规则沉淀`。
- **新增规则**：每次完成新功能/优化时，必须将改动特性同步更新写入项目根目录下的 `README.md`。
- **删除章节**：删除“项目概览”往后的所有架构与配置内容，替换为指向 `README.md` 的显式引用。

### 2.2 README.md 结构
- **引言**：增加“新增功能同步规则”的强约束提示。
- **功能特性**：合并 `AGENTS.md` 中有价值的项目背景信息，增加“分布式高可用与可靠异步结算架构”章节（包含 Shard 哈希分片路由、Redis 短租约 Failover 接管、Outbox 开局广播、异步结算 RabbitMQ Durable 队列与 Settlement Worker）。
- **运行环境**：细化专用 Conda 环境 `hmp_ai` 与 Python 3.10.20 物理路径解释器的要求，补齐快速/全量 pytest 测试命令。
- **API 地图**：针对非 WebSocket 类的常规 REST 接口，采用“核心接口清单表格”整理呈现，包含功能、方法、端点、鉴权等级，并引导通过本地启动 `/docs` (Swagger) 查看详细参数字段以防过期；全量手写保留 WebSocket 长连接交互协议与事件示例。
