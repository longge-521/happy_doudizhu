# 全屏沉浸式控制台页面布局设计规格说明书

为了优化控制台页面在大屏幕/高分辨率下的空间利用率，提升“审计日志”12列宽表格与“WebSocket调试”终端日志在垂直/水平方向的可读性，我们需要对前端进行方案 A（全屏平铺）改造。

## 设计目标
1. 将控制台容器扩展至浏览器视口大小（`100vw` * `100vh`），使页面内容整体平铺呈现，最大化降低大屏幕两侧的背景留白。
2. 将滚动行为限制在右侧的操作面板区内部，防止整个浏览器视口出现滚动条。
3. 优化卡片布局与滚动高度分配，针对大屏设备将“调试终端日志”框等局限性组件改造为弹性伸缩。

## 页面样式重构设计

### 1. 视口全屏化 (static/css/styles.css)
- **`body`**
  - 高度设为 `100vh`，宽度设为 `100vw`。
  - 内边距 `padding` 设为 `0`。
  - 设为 `overflow: hidden;`，强行禁锢整个视口的滚动条。
- **`.dashboard`**
  - 改为 `width: 100%; height: 100%;`。
  - 圆角 `border-radius` 修改为 `0`。
  - 边框 `border: none;`。
  - 外阴影 `box-shadow: none;`。
  - 内边距 `padding: 0;`。
  - 侧边栏的栅格宽度适当变更为 `260px 1fr`。

### 2. 局部溢出滚动 (static/css/styles.css)
- **`.sidebar`**
  - 高度 `100%`。
  - 增加内边距 `padding: 24px;`。
- **`.content-area`**
  - 高度 `100%`。
  - 增加内边距 `padding: 24px;`。
  - 开启 `overflow-y: auto;`。为了拥有更出色的页面体验，需要对它的滚动条进行美化，保证暗黑模式下的呼吸感。

### 3. 面板大屏自适应 (static/css/styles.css)
- **面板基础样式 `.panel`**
  - 高度设为 `100%`，使得它的子卡片可以按 `flex` 或弹性方式向下生长。
- **WebSocket 终端 `.terminal`**
  - 从固定高度 `height: 380px` 变更为弹性高度 `flex: 1`。这样可以在大屏中自动填充页面底部，扩展至最大可视视野。
- **大文件上传文件列表 & 审计日志表格容器 `.table-container`**
  - 确保拥有圆润的背景和内边距，并利用 `overflow-x: auto` 支持极端窄分辨率下的横向滚动。

### 4. 静态资源 Cache Buster (templates/index.html)
- 更改 HTML 尾部加载 console.js 及 styles.css 链接的版本号，采用 `?v=202606230240`，强行刷新客户端缓存。
