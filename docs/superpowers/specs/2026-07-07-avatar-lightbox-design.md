# 头像大图预览 (Lightbox) 与小挂件操作设计规约 (2026-07-07)

本规约旨在解决当前个人资料中头像交互手段单一的问题（仅有点击上传，缺失了用户查看原图/大图的诉求）。

## 目标与交互演进

- **原交互**：悬停在圆形头像上时全铺蒙版显示“📷 上传”，点击触发上传。无法查看大图。
- **新交互 (挂件化与大图分流)**：
  1. 圆形头像本身只读。点击头像本身时，触发全屏/大图预览弹窗（Lightbox），供用户欣赏或下载原图。
  2. 头像圆圈的右下角放置一个绝对定位的精致小圆形挂件按钮（包含 📷 相机图标），悬浮在其上时有微动效。点击该小挂件才触发本地图片选择和上传。
  3. 当显示头像预览弹窗时，采用高级的高斯模糊（Glassmorphism）暗色背景，中心展示大图，支持点击任意空白处关闭大图预览。

### 1. 头像容器样式调整
- `.profile-avatar-preview` 保留圆形及金色边框。
- 取消全局 Hover 黑色蒙层 `.avatar-hover-overlay`。
- 点击时绑定 `@click="handleViewLargeAvatar"`。

### 2. 挂件按钮 `.avatar-upload-badge` 样式
- **定位**：`position: absolute; right: 0; bottom: 0;`
- **尺寸**：约 `24px * 24px` 的圆形。
- **背景**：深色带有一点半透明（如 `rgba(0, 0, 0, 0.75)` 或金色 `background: #ffd700`，字色为深色）。
- **图标**：`📷` 或 SVG 相机图标，居中对齐。
- **层级**：`z-index: 10`。
- **动效**：`hover` 时放大 `transform: scale(1.1);` 并加深投影，增强点击引导性。

### 3. 大图灯箱预览 `.avatar-lightbox`
- **定位**：`position: fixed; top: 0; left: 0; width: 100vw; height: 100vh;`
- **层级**：`z-index: 9999`（确保浮在最上层）。
- **背景**：磨砂高斯模糊背景 `backdrop-filter: blur(10px); background: rgba(0,0,0,0.8);`。
- **图片尺寸**：宽度/高度自适应最大 `min(400px, 80vw)`，带圆角和金色发光阴影效果。
- **交互**：点击背景任意处均可退出。

## 验证计划

1. **基本布局检验**：确认右下角挂件按钮是否精准地定位在头像右下方，且没有发生像素溢出遮挡。
2. **大图弹窗动作检验**：点击头像大图，大图正确展示；且再次点击大图或暗色模糊背景能顺利关闭。
3. **上传功能检验**：点击右下角小相机，仍能正常弹出文件系统并替换预览头像。
