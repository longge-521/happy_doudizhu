# 510K 信息集一致 rollout 设计

## 根因证据

扩大到 8000 局、6000 局 rollout 和 8 个候选后，固定 500 局检查点一度达到 40.4%，但独立 1000 局仅 33.8%；同一牌集上的 teacher 为 35.0%。当前 rollout 从服务器完整房间复制局面，模拟时保留了两家真实暗牌，而模型输入只包含玩家可见信息。候选价值因此被不可见暗牌强烈影响，单次标签方差大，并与线上信息集不一致。

## 方案

- 观察者手牌、全部公开已出牌、当前牌墩、历史、剩余张数和得分保持不变。
- 将所有未见牌按两名对手公开的剩余张数随机重新分配，不使用真实对手手牌。
- 同一决策的全部候选共享相同的随机分配，减少候选间比较方差。
- 默认生成 2 个独立确定化局面，每个候选分别模拟到终局后取平均奖励。
- 第一轮使用 4 个候选控制耗时；确定化次数通过 CLI 配置。

## 安全与边界

- 只修改训练标签生成，不改变线上协议、模型输入和真人规则。
- 确定化必须保持观察者手牌、对手剩余张数、54 张牌唯一性和原房间不变。
- 所有随机数由对局种子派生，固定种子可复现。
- 训练/评测仍使用合法动作生成器，非法率门禁不变。

## 验证

- 测试确定化不保留真实暗牌、保持张数和全牌集合、固定种子可复现。
- 测试原始房间不被修改。
- 运行训练器与全部 510K 回归。
- 先做 4000 局、2 次确定化、4 候选短测；达到门槛后再进入 20000 局。
