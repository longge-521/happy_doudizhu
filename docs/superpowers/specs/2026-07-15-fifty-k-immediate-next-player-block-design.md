# 510K 立即下家听牌阻断设计

## 问题与根因

在三人各自为战的 510K 对局中，出牌顺序为“上一位玩家 -> 当前机器人 -> 下一位玩家”。当下一位玩家只剩一张牌、上一位玩家打出单张时，当前机器人如果不出或只用较小单牌跟牌，下一位玩家可能立即出完，另外两家同时失败。

现有模型能够看到相对座次的公开剩余张数，但线上排序没有不可违反的立即输局安全约束。残局搜索只检查模型前两个候选并只做一次暗牌确定化，可能把随机样本中的偶然安全误判为真实安全，也可能完全没有评估最大单张或特殊牌。

## 方案选择

采用推理层确定性战术门禁，不重新训练现有模型：

1. 仅当 510K 模式、立即下家剩一张、当前正在跟单张时触发。
2. 当前机器人能够直接出完时，直接获胜始终优先。
3. 从全部合法动作而不是模型 Top-2 中寻找阻断动作。
4. 优先选择下一位玩家用一张牌无法压制的动作：
   - 单张已达到公开未见牌的最高上界时，优先使用该单张；
   - 否则使用最低成本的合法多张特殊牌完成保证阻断。
5. 没有保证阻断动作时，选择最大合法单张，最大化阻断概率。
6. 没有任何可压动作时允许不出。
7. 桌面为对子、顺子等多张牌型时，单张听牌玩家本来无法跟牌，不触发该门禁。
8. 只剩一张的玩家不是立即下家时，不触发硬门禁，继续使用模型、规则排序和残局搜索。

该门禁只使用手牌、已出牌、座次和公开剩余张数，不读取对手隐藏手牌。

## 数据流

`ai_rank_play_candidates` 先生成全部合法动作并完成模型排序；随后根据 `AIContext.player_ids` 和 `ai_id` 确定立即下家。命中听牌风险时，由战术门禁从全部合法动作选择安全动作并固定为 Top-1，同时跳过可能重新覆盖该动作的随机残局搜索。未命中时保持当前逻辑不变。

## 测试与验收

- 真人出单 4，当前机器人有多张可压单牌，立即下家剩一张：Top-1 必须是最大安全单张，不能不出或小压。
- 模型把不出/小牌排在前面时，门禁仍覆盖模型 Top-1。
- 普通单张无法保证阻断而存在炸弹或 510K 时，使用最低成本特殊牌。
- 无可压动作时允许不出。
- 桌面为多张牌型、听牌者不是立即下家、下一家剩两张时均不误触发。
- 交换对手隐藏牌但保持公开信息相同，门禁结果不变。
- 运行 510K 战术、模型、残局搜索定向测试，并更新 README。

## 训练影响

本修复不改变特征版本、网络结构或权重格式，因此不需要重新训练。该场景应加入后续训练固定战术评测集，但不阻塞本次线上修复。
