# 510K 训练局中胜率日志实施计划

> **供后续执行者使用：** 逐项按测试先行完成，实施后运行定向测试并检查差异。

**目标：** 在 510K 训练的每批进度日志中显示模型对两名规则 AI 的即时与累计胜率，不增加额外评测对局。

**方案：** 采样单局在结束时返回两类数据：用于梯度更新的样本，以及仅限“一名模型、两名规则 AI”对局的胜负统计。采样批次合并统计，训练器在原有进度日志内输出本批和累计胜局/总局数；纯自博弈对局不纳入该指标。2000 局检查点评测和上线门槛保持不变。

**技术：** Python 3、PyTorch、pytest、`concurrent.futures.ProcessPoolExecutor`。

## 约束

- 不改变现有训练参数、模型权重、奖励公式或正式评测逻辑。
- 不新增额外对局，不影响训练速度。
- Windows 多进程返回的数据必须可序列化。
- README 同步说明日志字段。
